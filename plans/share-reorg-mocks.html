On “drop the many-sites idea” — the case for D

The scorecard is an image with no embedded link. X / Reddit / Facebook web-intents are link-only — they physically can't upload the card; they'd show the page's OG image, not the run. So the link buttons never actually carried the thing worth sharing.

On mobile, the native share sheet already exposes every app the player has (WhatsApp, Instagram, Messages, X…) — so a dedicated button row is redundant there. The only thing D gives up is desktop one-click-to-X/Reddit, and that was only ever sharing the generic site link + text.

My lean: D for the endgame (image-first, the menu we already have) — it matches “the card is the important thing” and drops dead weight. Optionally keep one “Copy link” in D for the desktop case. Keep the link buttons only on the site share (sharing the catalogue URL is genuinely a link, not a card).

Site share (footer + drawer) is unchanged in these mocks — it shares komyo.online as a link, so X / Facebook / Reddit / WhatsApp / Copy still make sense there. Open question: do we also want a brand/catalogue card image for the site, or keep it link-only?
