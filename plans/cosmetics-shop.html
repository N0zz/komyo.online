komyo — Challenge Shop

design report · 2026-07-03 · release priority #1 · rev 2
✨ Rev 2 — what changed after your review (all marked with dashed gold boxes below): currency is the Trophy 🏆 everywhere (every ◈ price replaced), shop named "Cosmetics" (never "Shop" — that's merch), trophies/cosmetics pill wording, favorite games sort to the top of the store, good-run trickle approved, bird prices reordered (Raven is #2 after Phoenix), and all five open questions resolved into decisions.

0 · Decisions locked in this session

  • Titles stay automatic — they keep unlocking from lifetime points exactly as today; the shop shows the ladder read-only as the prestige track. Spending never touches your title.
  • Two metrics: lifetime points (only ever grows; drives titles) and spendable balance = lifetime − spent. At launch, balance = full lifetime (nothing was ever spent).
  • Meadow Flyer birds migrate to challenge points, buy-any-order; in-game banked currency is removed — already-owned birds stay owned, remaining banked points convert 1:1 into challenge points (a one-time migration like the ×10 one).
  • Pricing = bands, not a total pool (catalogue size is unknown): cheap ≈ a daily (🏆 10–25), mid ≈ a few days (🏆 50), premium ≈ a week (🏆 100). Progressive exceptions (top birds, one "crown jewel" per game) may go above — but nothing should demand ~8 weeks of farming; hard ceiling 🏆 500.
  • Naming convention (locked 2026-07-03): the currency is the Trophy 🏆 — used everywhere prices, balances and earnings appear (no second symbol). The shop is called "Cosmetics" — never "Shop"/"Store", which stays reserved for future merch in the main ☰ menu. Player-facing copy renames "challenge points" → trophies across the drawer, challenge cards ("+10 🏆"), profile ("🏆 TROPHIES"), and the titles ladder. Asteroids+'s in-run ◈ credits are a different, per-run thing and keep their own symbol.
  • Good-run trophy trickle — approved, ships with the shop: +2 🏆 per good run, capped at 3/day (max +42/wk). Ties earning to actually playing, softens the casual curve.
  • Placement answers: Cosmetics lives in the challenges menu only (main ☰ reserved for merch) · selection is per-device, carried by Export/Import · bird price tail approved (tune later) · in-game STYLE groups go in the start menu · the collection bar counts purchasables only (titles have their own ladder).

1 · Economy

Earn rates today: daily 10 + weekly 50 → perfect week = 120, active ≈ 100, casual ≈ 40–60. One daily = one cheap item; one full week ≈ one premium item. Each game's launch set costs 🏆 200–400 total ≈ 2–4 weeks of active play per game — so "finish your favorite game's set" is quick, "chase 100%" scales with the catalogue (by design, per the bands decision).

Titles thresholds — reviewed, keep as-is (for now)

Ladder timing at active pace (~100/wk lifetime): Peasant 100 ≈ 1 wk · Squire 250 ≈ 2–3 wk · Knight 500 ≈ 5 wk · Sorcerer 1 000 ≈ 2–3 mo · Archmage 2 000 ≈ 5 mo · Oracle 4 000 ≈ 10 mo · Ascendant 6 500 ≈ 16 mo · Emperor 10 000 ≈ 2 yrs. Since titles key off lifetime, shop spending doesn't slow the ladder — the two tracks are fully independent, which is exactly why the pacing still works. The early tiers land satisfyingly fast, the tail is a real long-game. Recommendation: leave thresholds; retune from GA4 once real completion data flows (the existing challenge_done events already measure earn rate).

Earning headroom — APPROVED, ships with the shop✨ REV 2

+2 🏆 per good run, capped at 3/day (max +42/wk; realistically +20–30 for active players). Ties earning to actually playing — cheap to add since goodRuns are already tracked per day. It also raises the title pace slightly (lifetime grows faster); the thresholds review below already absorbs that. Parked streak bonuses could slot here later if more headroom is ever needed.

✨ Where the player sees the cap: (1) the end-of-run menu's existing "✓ Good run" line becomes the receipt — "✓ Good run · +2 🏆 (2/3 today)", and when maxed "✓ Good run · daily bonus maxed 3/3" — you learn the cap the moment it matters; (2) an always-on line in the challenges panel/drawer under the trophy pills — "⚡ Good-run bonus: 2/3 today · +2 🏆 each" — for planning "is one more run worth it?"; (3) one sentence in the FAQ's challenges entry. No new storage: the counter is min(goodRuns, 3) from the existing daily log.

7 · Data & implementation shape

  • cosmetics.js — new shared registry, loaded like challenges.js (both catalogue AND games; added to every SW SHELL in lockstep). Per item: { id: 'snake.food.gold', game, set, name, desc, price, painter } — the painter is a tiny canvas swatch (same idiom as the shop icons), so the modal previews without loading game code. Version field from day one.
  • Storage: gamekit_owned = { itemId: { c: cost, t: day } }. Spendable = lifetime(gamekit_done) − Σ owned costs — derived, never stored, so it can't drift. Defaults are implicit (never stored). Kept by Export/Import; per-game reset does NOT touch it (site-level purchase); Reset-all wipes it with everything else.
  • Kit API: gamekit.cosmetics → lifetime(), balance(), owned(id), buy(id) (validates balance, idempotent), selected(set)/select(set, id) (per-set choice, stored as gamekit_cos_sel), progress(game?) → {owned, total, pct}. Buying fires a GA4 shop_buy {id} (consent-gated) so prices can be tuned from data.
  • Shop modal = kit-owned (gamekit.shopPanel()), reusing the gkm shop-cell machinery (locked/price/two-step/focdesc). Opened from the challenges drawer pill + profile bar.
  • In-game: each game adds a STYLE grid group (or extends an existing selector like Flyer's) with locked+price cells; buying in place calls cosmetics.buy. Games apply the selected cosmetic in their render (a small COS = KIT.cosmetics.selected('snake.food') read).
  • Flappy migration: one-time — birds owned under the old ladder → written into gamekit_owned at cost 0; remaining banked points added to lifetime via a synthetic gamekit_done entry ({'flappy-migrate': N} — keeps spendable = lifetime − spent consistent); old keys removed; flagged like gamekit_pts_x10.
  • ✨ Trophy rename pass: player-facing "points"/"pts" → trophies / 🏆 everywhere — drawer pills, challenge card "+10 🏆" badges, profile "🏆 TROPHIES" meta, titles-ladder "pts" labels, FAQ + changelog wording. Internal storage keys (gamekit_done etc.) keep their names — this is copy only.
  • Phases: P1 kit (cosmetics.js + API + shop modal + drawer/profile bars + trophy rename + good-run trickle) with 2 games' sets (snake + breakout) → P2 flappy migration + asteroids×2 + KD → P3 the rest + cursors. Tests: buy/balance/idempotency, migration, registry-integrity (unique ids, every set has a free default), progress math, trickle cap.

8 · Extra ideas (mine — take or leave)

  • Collector milestones: at 25 / 50 / 100% collection, the profile gets a small collector badge next to the title (pure client-side, rides the collection bar). Makes the % chase itself rewarding.
  • Deed-locked exclusives (later): 1–2 items per game that money can't buy — unlocked by a feat ("score 500 in Neon Snake"), shown in the shop with the deed as the price. A second chase axis that keeps skilled players hunting. Cheap to add on top of the same registry (deed: instead of price:).
  • Live preview before buying: selecting a locked item temporarily applies it in the menu preview/backdrop (never in a recorded run). The two-step select state makes this nearly free.
  • NOT recommending: daily deals/discount rotations (FOMO mechanics clash with the no-predatory-patterns stance the challenges system already commits to) and any real-money anything, obviously.

Structural findings (beyond individual numbers)

  • Hard-cluster in the rotation: the 22-day daily list is ordered easy-half → hard-half, so days ~14–22 are a straight run of tier-2 goals. Suggest interleaving when targets get retuned.
  • Mode-agnostic targets leak: any mode counts — Stack's Zen trivializes its goals, Range's 60 s mode halves the 600 difficulty. Either accept (casual-friendly) or pin modes/durations per goal.
  • KD pair is redundant: wave-5 and score-700 are the same run on medium (and 700 arrives by ~wave 4 on Hard's ×2 multiplier). One of them should become a genuinely different ask.
  • Asteroids has one goal, everyone else two — add a second tier (e.g. 8,000) for rotation symmetry.
  • Candidate retunes to test (not applied): asteroids-plus 50k/150k → 12,000 / 40,000 (≈ wave 5 boss / wave 10 — matches other games' effort); stacker bar+goal 12 → 15; bubbles 3,000 → 2,500.
